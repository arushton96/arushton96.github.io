# Personal Projects

This section contains a collection of self-directed and workflow-driven automation projects focused on practical problem solving, process optimization, and experimentation with computer vision, scripting, and operational automation.

Unlike the larger academic and industrial systems elsewhere in this portfolio, these projects originated from real repetitive tasks, workflow inefficiencies, or personal experimentation ideas that evolved into fully functional automation tools.

While these projects are generally smaller in scope than the larger robotics and automation systems elsewhere in this portfolio, they demonstrate an important engineering mindset:

```text
Identify repetitive work
        ↓
Analyze deterministic patterns
        ↓
Design reliable automation
        ↓
Reduce manual intervention
```

The projects in this section emphasize:

- Practical automation design
- Workflow optimization
- Human-in-the-loop automation
- Computer vision experimentation
- Reliability-focused scripting
- Operational process improvement
- Python-based automation tools

## Projects

A collection of Python-based workflow automation tools developed to reduce repetitive operational processing tasks within shipping, logistics, and estimating workflows.

These tools focused on converting highly repetitive manual workflows into structured automated pipelines while preserving human oversight where operational judgment remained important.

Projects included:

- Automated medical-equipment invoice parsing
- PDF and email attachment processing
- Persistent item-reference database generation
- WhatsApp export parsing
- Automated estimate-document generation
- Structured workflow formatting tools

#### Technical Highlights

- PDF text extraction
- Regex-based parsing
- Automated spreadsheet generation
- Structured document generation
- Email attachment processing
- Human-in-the-loop workflow automation
- Persistent reusable operational data

#### Workflow Focus

### Computer Vision & Input Automation

A computer-vision-based automation project developed as a personal experimentation exercise focused on real-time image processing, event detection, and automated input control.

The project continuously monitored a defined screen region using OpenCV, detected visual splash events through contour analysis and threshold filtering, and automatically responded using simulated mouse and keyboard input.

Although intentionally lighthearted in application, the project implemented many practical automation concepts including:

- Region-of-interest monitoring
- Real-time event detection
- Contour analysis
- Timing/state management
- Reliability optimization
- Automated recovery handling

#### Technical Highlights

- OpenCV image processing
- Threshold filtering
- Contour extraction
- Event-driven automation
- Real-time screen capture
- Automated cursor control
- Reliability-focused automation tuning

#### Detection Pipeline

```text
Screen Capture
        ↓
Image Filtering
        ↓
Contour Detection
        ↓
Event Recognition
        ↓
Automated Input Response
```

## Engineering Approach

The projects in this section share a common engineering philosophy:

- automate repetitive deterministic work
- preserve human judgment where necessary
- prioritize reliability over unnecessary complexity
- design systems around real operational use cases

Rather than focusing on theoretical exercises, these projects were built around practical problems encountered during day-to-day workflows or personal experimentation.

Together, they demonstrate a broader interest in automation engineering beyond formal coursework and industrial systems, including scripting, process analysis, computer vision, and workflow optimization through software design.
